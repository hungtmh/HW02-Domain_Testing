# BUG-004: API không xác thực độ mạnh mật khẩu

## Metadata
| Field | Value |
|-------|-------|
| Feature | FR-01 |
| Severity | Major |
| Priority | High |
| Component | API |
| Test Case | BV-01, BV-04, BV-07, BV-08 |
| Status | Open |

## SRS Reference
> **Yêu cầu mật khẩu mạnh:** Tối thiểu 8 ký tự, có ít nhất 1 chữ hoa, 1 chữ thường, 1 chữ số và 1 ký tự đặc biệt (`@`, `$`, `!`, `%`, `*`, `?`, `&`).

## Environment
- OS: Windows 10
- Backend: localhost:3000

## Steps to Reproduce
1. Gửi request đăng ký với mật khẩu cực yếu (ví dụ: `Test12!` chỉ có 7 ký tự hoặc `test1234!` không có chữ hoa hoặc `Test1234` không có ký tự đặc biệt).
```powershell
Invoke-RestMethod -Uri "http://localhost:3000/api/register" -Method Post -ContentType "application/json" -Body '{"name":"Nguyen Van A","email":"new_auth_weak@test.com","password":"Test12!"}'
```

## Expected Result (SRS)
API từ chối đăng ký và trả về lỗi validate mật khẩu yếu (HTTP 400 Bad Request).

## Actual Result
Đăng ký thành công với HTTP 200 OK. Mật khẩu yếu vẫn được ghi nhận vào database.

## Evidence
- Bản ghi mới được lưu với mật khẩu `"Test12!"` (7 ký tự).
- Xem log thực thi tại [FR-01_test-execution.md](file:///d:/Kiem_thu/HW2/HW02-Group08/23127195/reports/FR-01_test-execution.md#L30).

## Notes
- Nguyên nhân: Thiếu hoàn toàn lớp validate mật khẩu ở Backend [server.js](file:///d:/Kiem_thu/HW2/HW02-Group08/backend/server.js). Do đó, người dùng có thể bypass qua giao diện Frontend và gửi trực tiếp request mật khẩu yếu lên API.
