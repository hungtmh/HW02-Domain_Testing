# BUG-003: API cho phép đăng ký khi bỏ trống các thông tin bắt buộc

## Metadata
| Field | Value |
|-------|-------|
| Feature | FR-01 |
| Severity | Critical |
| Priority | High |
| Component | API / Database |
| Test Case | DT-04, DT-05, BV-10 |
| Status | Open |

## SRS Reference
> Người dùng phải cung cấp: **Họ Tên**, **Email**, **Mật khẩu**.

## Environment
- OS: Windows 10
- Backend: localhost:3000

## Steps to Reproduce
1. Gửi request `POST` đăng ký với body rỗng `{}` (không cung cấp `name`, `email`, hoặc `password`).
```powershell
Invoke-RestMethod -Uri "http://localhost:3000/api/register" -Method Post -ContentType "application/json" -Body '{}'
```

## Expected Result (SRS)
API trả về lỗi yêu cầu điền đầy đủ thông tin (HTTP 400 Bad Request) và không lưu bản ghi rỗng vào cơ sở dữ liệu.

## Actual Result
API đăng ký thành công, trả về HTTP 200 OK:
```json
{
  "message": "User registered successfully",
  "id": 6
}
```

## Evidence
- Bản ghi mới trong bảng `users` được tạo với các giá trị cột `name`, `email`, `password` đều là `NULL`.
- Xem thêm log thực thi tại [FR-01_test-execution.md](file:///d:/Kiem_thu/HW2/HW02-Group08/23127195/reports/FR-01_test-execution.md#L29).

## Notes
- Nguyên nhân: [server.js](file:///d:/Kiem_thu/HW2/HW02-Group08/backend/server.js) không validate sự tồn tại của các tham số đầu vào.
- Cột `name`, `email`, `password` trong bảng `users` ở [database.js](file:///d:/Kiem_thu/HW2/HW02-Group08/backend/database.js) không khai báo thuộc tính ràng buộc `NOT NULL`.
