# BUG-007: Ô nhập Email sử dụng type="text" thay vì type="email"

## Metadata
| Field | Value |
|-------|-------|
| Feature | FR-01 |
| Severity | Minor |
| Priority | Low |
| Component | Web |
| Test Case | DT-04 (UI) |
| Status | Open |

## SRS Reference
> Phù hợp với chuẩn thiết kế giao diện (Email input type phải là email).

## Environment
- OS: Windows 10
- Browser: Chrome
- Frontend Web: localhost:5173/register

## Steps to Reproduce
1. Mở trang đăng ký tài khoản `http://localhost:5173/register`.
2. Chuột phải vào ô nhập Email và chọn "Inspect" (Kiểm tra phần tử).

## Expected Result (SRS)
Thẻ input dành cho Email phải được định dạng kiểu email:
```html
<input type="email" ... />
```

## Actual Result
Thẻ input dành cho Email có thuộc tính:
```html
<input type="text" ... />
```

## Evidence
- Mã nguồn tại [Register.jsx dòng 48](file:///d:/Kiem_thu/HW2/HW02-Group08/frontend-web/src/pages/Register.jsx#L48):
```javascript
<input
  type="text"
  value={email}
  onChange={(e) => setEmail(e.target.value)}
  className="w-full border p-2 rounded"
  required
/>
```

## Notes
- Tác động: Sử dụng `type="text"` làm mất đi tính năng tự động kiểm tra định dạng email cơ bản của trình duyệt (HTML5 validation) trước khi submit, đồng thời làm mất trải nghiệm người dùng trên thiết bị di động (không hiện bàn phím tối ưu cho email với phím `@`).
