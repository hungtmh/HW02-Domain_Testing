# BUG-001: Cho phép đăng ký email trùng

## Metadata
| Field | Value |
|-------|-------|
| Feature | FR-01 |
| Severity | Major |
| Priority | High |
| Component | API / Database |
| Test Case | DT-02, BV-12 |
| Status | Open |

## SRS Reference
> Email phải có định dạng hợp lệ (`user@domain.com`) và là **duy nhất** trong hệ thống.

## Environment
- Backend: localhost:3000
- DB: SQLite (`backend/database.js`)

## Steps to Reproduce
1. Đăng ký user với email `test@eshop.com` (đã tồn tại trong seed data)
2. Gọi `POST /api/register` với cùng email, password bất kỳ

## Expected Result
HTTP 4xx, thông báo email đã được sử dụng.

## Actual Result
HTTP 200, `{"message":"User registered successfully","id":10}`

## Root Cause (phân tích code)
- Cột `email` trong bảng `users` **không** có ràng buộc UNIQUE (`database.js` L53)
- API `register` không kiểm tra trùng trước INSERT (`server.js` L20–30)
