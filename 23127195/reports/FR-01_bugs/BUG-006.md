# BUG-006: Regex validate mật khẩu sai — yêu cầu khoảng trắng thay vì ký tự đặc biệt

## Metadata
| Field | Value |
|-------|-------|
| Feature | FR-01 |
| Severity | Major |
| Priority | High |
| Component | Web — `Register.jsx` |
| Test Case | DT-07, DT-08, BV-08, BV-09 |
| Status | Open |

## SRS Reference
> Mật khẩu mạnh: Tối thiểu 8 ký tự, có ít nhất 1 chữ hoa, 1 chữ thường, 1 chữ số và 1 ký tự đặc biệt (`@`, `$`, `!`, `%`, `*`, `?`, `&`).

## Steps to Reproduce
1. Mở `http://localhost:5173/register`
2. Nhập name, email mới
3. Thử password `Test1234!` (hợp lệ theo SRS) → Submit
4. Thử password `Test 1234` (có space, không có special SRS) → Submit

## Expected Result
- `Test1234!` → đăng ký thành công
- `Test 1234` → từ chối (thiếu ký tự đặc biệt theo SRS)

## Actual Result
- `Test1234!` → lỗi "Mật khẩu quá yếu..."
- `Test 1234` → có thể pass validation client (regex yêu cầu `\s`)

## Root Cause
```javascript
// Register.jsx L15 — flawedStrongPasswordRegex
/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*\s)[A-Za-z\d\s]{8,}$/
```
Yêu cầu whitespace (`\s`) thay vì ký tự đặc biệt SRS.
