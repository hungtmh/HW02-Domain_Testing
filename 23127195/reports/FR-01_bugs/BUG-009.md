# BUG-009: Lưu trữ mật khẩu dưới dạng Plaintext trong cơ sở dữ liệu

## Metadata
| Field | Value |
|-------|-------|
| Feature | FR-01 / Security |
| Severity | Critical |
| Priority | High |
| Component | API / Database |
| Test Case | DT-01 / Code Review |
| Status | Open |

## SRS Reference
> Phù hợp với tiêu chuẩn bảo mật dữ liệu người dùng (Mật khẩu phải được mã hóa trước khi lưu trữ).

## Environment
- OS: Windows 10
- Backend: localhost:3000
- DB: SQLite (`backend/database.sqlite`)

## Steps to Reproduce
1. Đăng ký tài khoản thành công qua UI hoặc gọi API.
2. Mở cơ sở dữ liệu SQLite `database.sqlite` (ví dụ bằng DB Browser for SQLite) hoặc xem trực tiếp qua bảng `users`.
3. Kiểm tra nội dung của cột `password`.

## Expected Result (SRS)
Mật khẩu phải được băm/mã hóa (ví dụ bằng bcrypt) thành một chuỗi ký tự an toàn trước khi lưu trữ (ví dụ: `$2a$10$...`).

## Actual Result
Mật khẩu được lưu trữ dưới dạng văn bản thuần (plaintext) đúng y hệt giá trị người dùng nhập (ví dụ: `"Test1234!"`).

## Evidence
- Trong `backend/database.js` dòng 92:
```javascript
insertUser.run('Admin User', 'admin@eshop.com', 'Admin123!', 'admin');
insertUser.run('Test User', 'test@eshop.com', 'Test1234!', 'user');
```
Mật khẩu mẫu của Admin và Test User được lưu trực tiếp bằng plaintext.
- Trong `backend/server.js` dòng 22–29:
```javascript
db.run(
  "INSERT INTO users (name, email, password) VALUES (?, ?, ?)",
  [name, email, password],
  ...
)
```
API ghi nhận trực tiếp giá trị `password` nhận được từ client mà không qua bất kỳ hàm băm nào.

## Notes
- Đây là lỗi bảo mật vô cùng nghiêm trọng. Nếu database bị rò rỉ, toàn bộ tài khoản và mật khẩu của người dùng sẽ bị lộ.
- Khắc phục: Sử dụng thư viện `bcryptjs` để hash mật khẩu trước khi INSERT, và dùng `bcrypt.compare` để đối chiếu trong API Login.
