# BUG-002: API cho phép đăng ký email sai định dạng

## Metadata
| Field | Value |
|-------|-------|
| Feature | FR-01 |
| Severity | Major |
| Priority | High |
| Component | API |
| Test Case | DT-03, BV-12 |
| Status | Open |

## SRS Reference
> Email phải có định dạng hợp lệ (`user@domain.com`) và là **duy nhất** trong hệ thống.

## Environment
- OS: Windows 10
- Backend: localhost:3000

## Steps to Reproduce
1. Gửi request `POST` đăng ký người dùng với email không có định dạng đúng (thiếu `@` hoặc sai cấu trúc domain).
```powershell
Invoke-RestMethod -Uri "http://localhost:3000/api/register" -Method Post -ContentType "application/json" -Body '{"name":"Nguyen Van A","email":"invalid-email","password":"Test1234!"}'
```

## Expected Result (SRS)
API trả về lỗi xác thực đầu vào (ví dụ: HTTP 400 Bad Request) và từ chối ghi nhận vào cơ sở dữ liệu.

## Actual Result
API đăng ký thành công, trả về HTTP 200 OK:
```json
{
  "message": "User registered successfully",
  "id": 5
}
```

## Evidence
- Bản ghi lưu vào DB chứa cột email có giá trị `"invalid-email"`.
- Xem thêm log thực thi tại [FR-01_test-execution.md](file:///d:/Kiem_thu/HW2/HW02-Group08/23127195/reports/FR-01_test-execution.md#L28).

## Notes
- Nguyên nhân: API `POST /api/register` trong [server.js](file:///d:/Kiem_thu/HW2/HW02-Group08/backend/server.js) không có logic validate định dạng chuỗi email (bằng regex hoặc thư viện như validator).
